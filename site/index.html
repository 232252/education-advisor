为何不同

"AI + 教育"产品很多，
但这一个是真的

我们不是在聊天框外面套个壳，也不是把学生数据送到云端。下面五个差异，是工程上的硬承诺。

01

真·本地优先，Rust 主干

学生数据的每一个字节都存在本地 Rust 事件存储里。LLM 是唯一触网的部分，且只拿到必要的数据。Rust CLI 负责全部读写、校验、并发（文件锁）、原子持久化（tmp → fsync → rename）和 PII 加密。AI 层故意做成无状态可替换——换掉 GPT-4o 用 Qwen 4B 也能跑同一套代理、读同一份数据。

Rust CLI · eaa-cli
↑↓
Electron Main · Node 22
↑↓
React 18 · Vite · Tailwind
02

18 个协作代理，不是一个聊天框

class-monitor 记 +2/−3 操行分，risk-alert 关联 14 天趋势并预警，weekly-reporter 自动生成周五班级报告，validator 交叉核对数学是否对得上事件日志，psychology 只观察不写事件——这更像一套"教学团队操作系统"，而不是一个聊天机器人。

main governor counselor supervisor validator academic psychology safety home_school research executor bug-hunter class-monitor risk-alert data-analyst student-care discipline-officer weekly-reporter
03

"小模型规则书"——3B 也能跑得好

我们故意把每个代理的提示词设计成 Qwen 3.5 4B 也能工作的形态——6GB 显卡、本地推理、零成本。每个数字必须来自工具调用，每次写入必须授权，每个输出都用 JSON Schema 校验。详见 config/SMALL_MODEL_RULES.md 的 5 条铁律。

1数字必须来自工具调用
2每次写入必须授权
3输出必须符合 JSON Schema
4无工具，不输出
5所有决策可追溯
04

PII：可选、可逆、可审计

Rust 隐私引擎为每次安装构建一张加密映射表（"张小明" → S_017），暴露 11 个 IPC 操作：init、load、enable、disable、list、add、anonymize、deanonymize、filter（按收件人）、dryrun、backup。您可以把脱敏后的全班名单交给 LLM，输出报告时再"水合"回真名。

张小明→S_017
李雷→S_023
韩梅梅→S_041
🔒 AES-256-GCM
05

可复现构建，零惊喜

npm ci && npm run build && npm run package 在任何带 Node 22 的 Windows 机器上产出字节一致的安装包（除时间戳外）。没有隐藏的系统状态，没有不透明的原生模块——除了 better-sqlite3 和 Rust 编译的 EAA 二进制。完整供应链在这个仓库里。

npm ci
npm run build:eaa
npm run build
npm run package
→ Education Advisor-0.1.0-rc.1-Setup.exe

  1. 1

    克隆与安装

    git clone https://github.com/232252/education-advisor.git
cd education-advisor
npm ci

    需要 Node.js ≥ 22、npm ≥ 10、能编译 better-sqlite3 的 C++ 工具链。

  2. 2

    获取 Rust 后端

    npm run build:eaa

    下载对应平台的 eaa-cli 预编译二进制到 resources/eaa-binaries/，或自行从 core/eaa-cli/ 源码构建。

  3. 3

    开发模式

    npm run dev

    并行启动 Vite 主进程监听、Vite 渲染进程开发服务器（http://localhost:5173），新开一个终端跑 npm run dev:electron 启动 Electron 壳。

  4. 4

    打 release 包

    npm run build           # vite build × 2 configs
npm run package         # → release/Education Advisor-0.1.0-rc.1-Setup.exe
npm run package:portable

    Windows NSIS 安装包 / 单文件绿色版一键产出。macOS / Linux 改 electron-builder.yml 即可。

  5. 5

    首次配置

    打开应用进入 #/settings：选主题与语言 → 在 #/models 加至少一个 LLM API Key →（可选）配置飞书 / 初始化隐私引擎。然后到 #/agents 点 class-monitor 的"手动运行"，录入第一条操行事件。

常见问题

答你所问

这是只支持中文的产品吗？

UI 和数据（学生姓名、班级、学校）天然是中文。代码、提示词、开发者文档都是中英双语；代理提示词可编辑为任何语言。

不联网能用吗？

应用本身完全离线可用。只有 LLM 调用需要网络。如果您指向 localhost 上的 Ollama / LM Studio，整套系统可全离线运行。

可以不用 Rust 的 eaa-cli 吗？

可以。所有 EAA 调用都汇集在 src/main/services/eaa-bridge.ts 一个漏斗里。替换成 PostgreSQL / Firestore / 您自己的服务，代理提示词和 UI 完全不用改。

为什么用 Electron 不用 Tauri？

项目启动时，Windows 代码签名和自动更新在 Tauri 生态里还不成熟。我们在 ROADMAP 中长期跟踪 Tauri 生态。

安装包有多大？

Windows NSIS ≈ 85 MB / 绿色版 ≈ 75 MB。其中 Electron 运行时 ~50 MB、Rust EAA 二进制 ~20 MB，其余是您的代码、配置和 18 个代理。

怎么加一个新代理？

读 docs/AGENT_AUTHORING.md。TL;DR：在 agents/your-id/ 放一对 SOUL.md + AGENTS.md，在 config/agents.yaml 加一行，重启应用。

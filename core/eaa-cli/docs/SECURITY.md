# 安全规范

本系统为教育管理辅助工具，所有使用者和部署者必须遵守以下安全规范。

## 数据访问限制

1. **`eaa` CLI 是唯一数据读写入口**，禁止直接修改 JSON 文件
2. 新增事件必须通过 `eaa add` 命令，由 Rust 强制校验原因码和分值
3. 禁止绕过校验机制修改数据
4. 撤销操作必须通过 `eaa revert`，保留完整审计轨迹

## 🔒 隐私脱敏规范（v3.2 新增）

### 强制规则

**所有Agent在写入文件、发送到外部系统前，必须使用S_XXX化名，禁止包含学生真名。**

### 脱敏流程

```
1. eaa CLI 获取数据（含真名）
2. eaa privacy anonymize "含真名文本" → S_XXX版本
3. S_XXX版本写入本地JSON文件
4. 推送给教师 → eaa privacy deanonymize 还原真名
5. 发给外部AI → 直接用S_XXX版本
```

### 脱敏范围

| 字段 | 脱敏格式 | 示例 |
|:-----|:---------|:-----|
| 学生姓名 | S_XXX | S_024 |
| 身份证号 | 前6位+****+后4位 | 513324****1814 |
| 电话号码 | 前3位+****+后2位 | 136****18 |
| 家庭地址 | 只保留县+乡镇 | XX县XX镇 |

### 适用场景

| 场景 | 脱敏 | 说明 |
|:-----|:----:|:-----|
| 推送给教师本人 | ❌ 不脱敏 | 教师本人可见 |
| 发给其他系统/外部AI | ✅ 脱敏 | 用化名 |
| 本地JSON文件存储 | ✅ 脱敏 | 防止外发时泄露 |
| Agent间通信 | ✅ 脱敏 | 最小权限原则 |
| Cron推送摘要 | ✅ 脱敏 | 除非明确给教师 |

### 映射表安全

- 映射表使用AES-256-GCM加密存储
- 密码通过环境变量传入，不硬编码
- 52名学生全部映射（S_001~S_052）
- `eaa privacy list` 查看映射（需密码）

## 系统安全限制

1. 本系统不得用于操控宿主系统
2. 禁止通过本系统执行未经授权的系统命令
3. 禁止访问非教育数据相关的系统资源
4. AI Agent 仅限读写教育数据，不得越权

## 数据安全

1. 学生数据属于敏感信息，部署者需遵守相关法律法规
2. **禁止将真实学生数据上传至公开仓库**
3. 示例数据（`examples/` 目录）仅供测试使用
4. 凭证（API Key、Token 等）禁止明文存储
5. **GitHub仓库中不得包含任何Token/App Secret/用户ID**

## Agent 行为限制

1. Agent 只能通过 `eaa` CLI 读写数据
2. Agent 不得自主修改系统配置
3. Agent 不得执行未经授权的操作
4. 所有 Agent 行为必须有日志记录
5. **Agent写入的JSON文件必须使用S_XXX化名**

## 事件溯源安全

- 事件一旦写入不可删除，只能通过 REVERT 事件对冲
- 所有数据操作完全可追溯
- 支持与外部数据源交叉校验
- `eaa validate` 可随时校验全部事件完整性

## 定时任务安全

- 18个Cron任务全部包含隐私脱敏规则
- `delivery=announce` 的任务仅推送给教师本人（用真名）
- 其他任务输出使用S_XXX化名
- Validator定期审计Agent输出中的隐私合规性

## 发现的问题及修复

### 1. EAA Schema 缺失 (已修复)
- **问题**: addEvent 报 "系统找不到指定的路径", doctor 报 "原因码Schema缺失"
- **根因**: EAA bridge initialize() 未成功创建 schema/reason_codes.json
- **修复**: 手动创建 schema 文件, 转换 flat → nested 格式

### 2. deleteStudent 签名问题 (已修复)
- **问题**: deleteStudent('name',{confirm:true,reason:'x'}) 报 "reason must be a string"
- **根因**: preload 签名是 deleteStudent(name, reason?), 自动包装为 {confirm:true, reason}
- **修复**: 修正所有测试脚本使用 deleteStudent('name', 'reason string')

### 3. UI 渲染时序问题 (已修复)
- **问题**: 连续运行后 R55/R56/R57 仪表盘/学生页偶发渲染失败
- **根因**: React 异步渲染, 短暂等待不足以完成
- **修复**: 增加 UI 等待时间 + 重试逻辑 (1200ms + 1500ms retry)

### 4. 数据累积导致 range 截断 (已修复)
- **问题**: R55 事件数为 0, 但事件实际已创建
- **根因**: 多轮测试累积 1242 事件/536 学生, range(limit=1000) 截断新事件
- **修复**: 重置 EAA 数据目录 (符合用户"删除以前的测试数据"要求)

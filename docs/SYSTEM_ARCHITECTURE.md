## 二、Agent职责矩阵

| Agent | 核心职责 | 调度方式 | 输出 |
|:------|:---------|:---------|:-----|
| main | 消息路由、数据协调 | 实时 | 用户回复 |
| supervisor | 风险评估、报告生成 | 定时22:00 | 督导报告 |
| validator | 数据核验、质量控制 | 定时每6h | 核验结果 |
| academic | 学业分析、成绩预警 | 定时07:05 | 学业报告 |
| psychology | 心理监测、危机预警 | 定时21:00 | 心理评估 |
| safety | 安全检查、风险识别 | 定时周一 | 安全报告 |
| home_school | 家校沟通、通知生成 | 定时08:30 | 通知草稿 |
| research | 科研辅助、数据收集 | 定时22:10 | 论文观点 |
| executor | 系统维护、错误修复 | 定时01:00 | 维护报告 |
| talk_planner | 谈话计划、话术生成 | 按需触发 | 谈话计划 |

## 三、数据流转

所有数据流转都通过 `eaa` CLI，事件溯源贯穿全局：

```
用户消息 → main Agent → eaa CLI 写入事件 → 事件流追加
                                      ↓
supervisor 读取 ← eaa CLI 查询 ← 事件重放引擎
```

## 四、多通道支持

EAA 不绑定任何单一通信通道。OpenClaw 框架统一处理消息路由，Agent 只关注业务逻辑：

```
飞书消息  ─┐
QQ消息   ─┤
Discord  ─┼→ OpenClaw Gateway → main Agent → 业务处理
Telegram ─┤
微信     ─┘
```

## 五、扩展指南

### 新增Agent
1. 创建 `agents/{agent_name}/SOUL.md`
2. 在 `config/agents.yaml` 注册
3. 确保数据读写通过 `eaa` CLI

### 新增通道
无需修改任何 Agent 代码。只需在 OpenClaw 配置中启用新通道即可。

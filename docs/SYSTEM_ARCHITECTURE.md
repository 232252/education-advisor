## 二、Agent职责矩阵

| Agent | 核心职责 | 调度方式 | 输出 |
|:------|:---------|:---------|:-----|
| main | 消息路由、数据协调 | 实时 | 用户回复 |
| supervisor | 风险评估、报告生成 | 定时22:00 | 督导报告 |
| validator | 数据核验、质量控制 | 定时每6h | 核验结果 |
| academic | 学业分析、成绩预警 | 定时07:05 | 学业报告 |
| psychology | 心理监测、危机预警 | 定时21:00 | 心理评估 |
| safety | 安全检查、风险识别 | 定时周一 | 安全报告 |
| home_school | 家校沟通、通知生成 | 定时08:30 | 通知草稿 |
| research | 科研辅助、数据收集 | 定时22:10 | 论文观点 |
| executor | 系统维护、错误修复 | 定时01:00 | 维护报告 |
| talk_planner | 谈话计划、话术生成 | 按需触发 | 谈话计划 |

## 三、数据流转

### 3.1 学生数据流转

```
学生事件发生
    ↓
main Agent接收
    ↓
保存到inbox队列
    ↓
更新学生档案 (MD)
    ↓
同步到数据库 (JSON)
    ↓
通知supervisor更新风险
    ↓
纳入下次督导报告
```

### 3.2 定时任务数据流

```
Cron触发 (22:00)
    ↓
supervisor启动
    ↓
Python快速扫描 (3秒)
    ↓
生成风险报告
    ↓
写入归档文件
    ↓
main Agent汇总
    ↓
统一推送飞书
```

## 六、安全设计

### 凭证管理
- API密钥存储在 `config/app_config.json`（不提交到Git）
- 使用环境变量备份
- 定期轮换机制

### 数据隔离
- 每个Agent有独立工作区
- 共享数据通过统一接口
- 敏感信息脱敏处理

### 审计日志
- 所有操作记录到日志
- 错误自动上报
- 操作可追溯

## 七、扩展指南

### 新增Agent
1. 创建 `agents/{agent_name}/` 目录
2. 编写 `SOUL.md` 定义角色
3. 在 `config/agents.yaml` 注册
4. 分配工作区

### 新增技能
1. 在 `skills/` 创建技能目录
2. 编写 `SKILL.md` 定义接口
3. 在需要的地方引用

### 新增数据存储
1. 确定数据类型
2. 选择存储方式（MD/JSON/数据库）
3. 在数据层注册
4. 编写读写接口

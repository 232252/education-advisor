## 一、系统概述

CoPaw 事件溯源操行分系统 v2.0 是一套基于 **Event Sourcing（事件溯源）** 架构设计的操行分管理 CLI 工具，服务于四川省甘孜州九龙高级中学2024级5班（52名学生）的日常行为管理。系统使用 **Rust** 编写，以 JSON 文件作为持久化存储，通过事件重放（Replay）机制计算每个学生的实时操行分数。

### 1.1 核心设计理念

传统操行分系统采用"当前状态覆盖"模式——每次扣分直接修改分数，历史记录不可追溯。CoPaw v2.0 采用事件溯源架构，**所有状态变更均以不可变事件记录**，当前分数通过从初始状态重放全部事件计算得出。这带来以下核心优势：

- **完全可追溯**：任何时间点的分数都可以通过重放到该时刻的事件来还原
- **天然审计**：每一个分数变动都有对应的事件记录，无法篡改
- **可撤销**：通过添加 REVERT 事件来"撤销"之前的操作，而不是删除
- **数据一致性**：飞书 Bitable 中的数据可以随时与事件溯源重放结果对比校验

### 1.2 技术栈

| 组件 | 技术 | 说明 |
|:-----|:-----|:-----|
| CLI 核心引擎 | Rust 2021 Edition | 高性能编译型语言，零运行时开销 |
| CLI 框架 | clap v4 (derive) | Rust 生态最成熟的命令行参数解析库 |
| 序列化 | serde + serde_json | Rust 标准序列化框架 |
| 日期处理 | chrono v0.4 | 时间戳格式化与解析 |
| 错误处理 | thiserror v1 | 派生宏式错误类型定义 |
| 数据存储 | JSON 文件 | 轻量级，无需数据库，人类可读 |
| 数据迁移 | Python 3 | 从旧系统迁移历史数据 |

### 2.3 事件溯源核心机制

每个学生初始分数为 **100分**。系统从事件流中筛选该学生的所有事件，按时间顺序累加计算：

```
当前分数 = 100 + Σ(事件.分值)  // 对该学生的所有事件
```

事件一旦写入 `events.json`，不可修改、不可删除（只能通过 REVERT 事件抵消）。

## 六、CLI 命令手册

### 6.1 基础命令

```bash
copaw info                    # 显示系统基础信息（学生数、事件数）
copaw validate                # 验证所有事件数据的完整性
copaw stats                   # 显示详细统计（原因码分布、标签分布、分数区间）
copaw codes                   # 列出所有原因码及标准分值
```

### 6.2 查询命令

```bash
copaw score <姓名>            # 查询个人当前分数及风险等级
copaw history <姓名>          # 查询个人完整事件时间线
copaw ranking [数量]          # 排行榜（默认显示前20，可指定52显示全部）
copaw search <关键词>          # 搜索事件（支持姓名、描述、原因码）
copaw tag [标签名]            # 列出所有标签 / 按标签筛选事件
copaw range <开始日期> <结束日期>  # 按日期范围查询事件
```

### 6.3 系统命令

```bash
copaw replay                  # 重放全部事件，输出排行榜及变动值
```

### 6.4 使用示例

```bash
# 查看某学生分数
/vol2/copaw-data/copaw score 王勇

# 搜索所有睡觉事件
```
